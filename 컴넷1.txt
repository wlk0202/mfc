선 연결만 되서는 정보전달이 되지않는다.
네트워크가 2개면 네트워크주소도 2개 있어야한다.

dly- 딜레이
full duplex-쌍방향으로 전달하는것(동시에)
half duplex-송,수신선이 하나로 한쪽으로만 계속 주는것(동시x)(ex.방송국)  


ip주소와 서브넷 마스크를 and연산하여 나온 네트워크 주소가 같으면 같은 네트워크에 있음



ip주소 바꾸는것- no ip address ip주소 서브넷마스크

라우팅 테이블 - 라이팅이 되는 상황인가 확인

ping ip주소//설정된 주소로 핑을 날리는것
R1#sh ip int brief// 제대로 설정되었는지 확인하는것
R1#sh ip route//라우팅 테이블 확인

R2는 네트워크 2개를 연결하고 있어 ip를 두번 설정해야한다.
R1>enable
R1#config t

라우트안에 같은 라우팅테이블이 존재해야 정보전달이 된다.(핑가능)
ex)R1의 라우팅 테이블 안에 192.200.0.0이면 192.200.0.2로 핑 가능
   R1의 라우팅 테이블 안에 192.201.0.0이 없으면 192.201.0.2로 핑 불가능
